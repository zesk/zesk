#
# This test file format is simple: description line, blank line, test rrule, blank line, 
# Tests from: http://www.kanzaki.com/docs/ical/rrule.html, slightly modified
# 
# Date result format supports [DD,DD,DD,DD] or [SS-FF] where iteration by one between SS and FF
# The token "..." means forever
# The token "===>" on a line means stop processing that result (and skip to next blank line)
# 
Yearly in June and July for 10 occurrences:

DTSTART;TZID=US-Eastern:19970610T090000
RRULE:FREQ=YEARLY;COUNT=10;BYMONTH=6,7

[1997-2001]-[06,07]-10 09:00:00 EDT
==> (1997 9:00 AM EDT)June 10;July 10
(1998 9:00 AM EDT)June 10;July 10
(1999 9:00 AM EDT)June 10;July 10
(2000 9:00 AM EDT)June 10;July 10
(2001 9:00 AM EDT)June 10;July 10

#  Note: Since none of the BYDAY, BYMONTHDAY or BYYEARDAY components
#  are specified, the day is gotten from DTSTART
